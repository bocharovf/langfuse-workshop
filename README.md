# Cookbook Agent
Демо-приложение для воркшопа [Langfuse](https://langfuse.com/).

## Архитектура
```
┌───────────────────┐
│ cookbook-agent    ├─────────┐
└────────┬──────────┘         │
         │                    │
         │             ┌──────┴───────┐  ┌──────────────┐
         │             │  mcp-server  │  │  rag-loader  │
         │             └──────┬───────┘  └──────┬───────┘
         │                    │                 │
   ┌─────┴─────┐        ┌─────┴─────┐           │
   │   Redis   │        │  QDrant   ├───────────┘
   └───────────┘        └───────────┘
```

## Как подготовиться к воркшопу
Системные требования:
- Docker v24.x+
- Docker Compose v2.22.x+

Чтобы не терять время во время воркшопа подготовься заранее:
- Клонируй локально репозиторий
- Выполни следующую команду, чтобы скачать используемые Docker-образы
```shell
docker compose pull
```
- Выполни следующую команду, чтобы собрать Docker-образы компонентов проекта
```shell
docker compose build
```

Если всё прошло успешно, то ты готов к воркшопу!

## Как запустить проект
Подготовь `.env` файл:
- Скопируй `.env_example` в `.env`
- Заполни актуальные ключи Langfuse и ключ доступа к LLM

Запусти проект:
```shell
docker compose up
```

Когда в консоли появятся сообщения от `mcp-server` и `cookbook-agent` можно открывать веб-UI в браузере
по адресу http://localhost:8501

## Как изменять код
При изменениях кода в папке `cookbook-agent/app` Streamlit должен автоматически перезагружать веб-UI.

Чтобы применить изменения кода в папках `mcp-server/app` или `rag-loader/app` выполни следующую команду в новом терминале:
```shell
docker compose restart %component name%
```
Где `%component name%` либо `mcp-server`, либо `rag-loader`. Можно перечислять несколько компонентов в одной команде.
В основном терминале появятся логи от перезапуска этих компонентов.

В случае сбоев при обновлении Streamlit аналогично можно перезапустить и `cookbook-agent`:
```shell
docker compose restart cookbook-agent
```

## Как изменять параметры окружения (environment variables) / фича флаги (feature flags)
Конфигурация каждого компонента задана через параметры окружения (environment variables), фича-флаги тоже задаются как параметры окружения.

Изменение параметров окружения требует пересоздания компонента:
```shell
docker compose up --force-recreate -d %component name%
```

Где `%component name%` это:
- `cookbook-agent`
- `mcp-server`
- `rag-loader`
- `agent-setup`

Также можно перечислять несколько компонентов в одной команде.

## Как изменять промпт в Langfuse
Создай если нужной версии промпта ещё нет. После этого:
- Наведи курсор на нужную версию и нажми кнопку со стрелочкой.
- Укажи радиокнопку "Promote to production?" и затем save.

## Как изменять данные RAG
Инжестинг данных RAG не выполняется повторно при наличии существующей коллекции `recipes`

Сначала удаляем существующую коллекцию:
- Заходим в дэшборд QDRant - http://localhost:6333/dashboard
- Нажимаем на три точки в колонке "Actions" строки коллекции `recipes`
- Выбираем "Delete" в дропдауне и подтверждаем деструктивные действия в попапе

Для переинжеста просто перезапускаем `rag-loader` следующей командой в новом терминале:
```shell
docker compose restart rag-loader
```

Когда `rag-loader` завершит выполнение в дэшборде QDRant снова появится коллекция `recipes`.

## Как изменять зависимости
При изменении списка зависимостей в файлах `requirements.txt` выполни следующую команду в новом терминале:
```shell
docker compose up --build --force-recreate -d %component name%
```

Здесь `%component name%` как в пункте про изменение параметров окружения.

Такие же действия нужно проделать при изменении содержимого `Dockerfile`.
